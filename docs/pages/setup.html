How to Setup

Connect Fly Scoreboard to a Browser Source and point that source to the included overlay.

Step-by-step in OBS

  1. Create a Browser Source in your scene (Sources → + → Browser).
  2. Enable Local file and select the overlay index.html.
  3. Set a sensible width/height (example: 1920×1080, or match your canvas).
  4. Disable “Shutdown source when not visible” if you want timers to continue updating off-screen.
  5. Open the Fly Scoreboard dock and select the Browser Source used for the overlay.

Recommended Browser Source settings

  • Custom CSS: leave empty (use your overlay style.css instead).
  • Refresh cache of current page: useful after editing files.
  • FPS: 30 is usually enough; 60 if you animate heavily.
The overlay runtime fetches plugin.json with cache: "no-store" once per second. If your overlay looks “stuck”, force-refresh the Browser Source.

Folder placement rule

In the default overlay runtime (script.js), the state file is loaded as: 

fetch("plugin.json", { cache: "no-store" })

That means plugin.json must be reachable at the same relative path as index.html.

Common troubleshooting

  • Blank overlay: confirm the Browser Source is pointing to the correct index.html.
  • No updates: ensure plugin.json is in the same folder, and refresh the Browser Source.
  • Wrong teams left/right: check the swap_sides toggle (template uses team_x/team_y).
