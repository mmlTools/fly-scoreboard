How to Setup

Connect Fly Scoreboard to a Browser Source and point that source to the included overlay or one of your template folders.

Step-by-step in OBS

  1. Create a Browser Source in your scene (Sources → + → Browser).
  2. Enable Local file and select the overlay index.html.
  3. Set a sensible width/height (example: 1920×1080, or match your canvas).
  4. Disable “Shutdown source when not visible” if you want timers to continue updating off-screen.
  5. Open the Fly Scoreboard dock and select the Browser Source used for the overlay.

Recommended Browser Source settings

  • Custom CSS: leave empty (use your overlay style.css instead).
  • Refresh cache of current page: useful after editing files.
  • FPS: 30 is usually enough; 60 if you animate heavily.
The overlay runtime connects to ws://127.0.0.1:4457 for instant updates. If the socket is unavailable, it fetches plugin.json with cache: "no-store" once per second.

Template folders

Use the dock Template row to select a parent folder that contains scoreboard templates. Each template is a subfolder with at least: 

My Templates/
  Soccer Lower Third/
    manifest.ini
    index.html
    style.css
    script.js

After choosing the parent folder, select a template from the dropdown. The plugin points the selected Browser Source at that template's index.html and creates plugin.json there if it is missing.

Each theme folder must include manifest.ini. The template dropdown reads the theme title and metadata from that file: 

title=Soccer Lower Third
author=Your Name
author_url=https://example.com
description=Compact lower-third scoreboard for soccer streams.
version=1.0.0

The selected template folder is the active resources path. Team logos, state updates, and Browser Source syncing all use that active template folder.

Folder placement rule

In the default overlay runtime (script.js), fallback state is loaded as: 

fetch("plugin.json", { cache: "no-store" })

That means plugin.json must be reachable at the same relative path as index.html.

WebSocket endpoint

The dock starts a local-only WebSocket server at: 

ws://127.0.0.1:4457

To use another endpoint in a custom Browser Source URL, pass it as a query parameter: 

index.html?ws=ws://127.0.0.1:4457

Common troubleshooting

  • Blank overlay: confirm the Browser Source is pointing to the correct index.html.
  • No updates: check the dock WebSocket status, ensure plugin.json is in the same folder, and refresh the Browser Source.
  • Wrong teams left/right: check the swap_sides toggle (template uses team_x/team_y).
