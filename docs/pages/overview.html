Overview

Fly Scoreboard is an OBS Studio plugin that broadcasts live scoreboard state over a local WebSocket, keeps a plugin.json fallback on disk, and ships a lightweight overlay runtime that renders your scoreboard by replacing template placeholders ({{…}}) in HTML.

What you get

  • A dock UI to manage teams, scores, timers, and single stats.
  • A ready-to-use overlay (index.html, style.css, script.js).
  • Local remote control through ws://127.0.0.1:4457.
  • A small templating engine: write {{home.title}}, {{timers[0].mmss}}, etc.

How it works

  1. The plugin updates state in memory and writes plugin.json.
  2. The overlay receives pushes from the local WebSocket, then falls back to polling plugin.json if needed.
  3. An animation loop renders timers smoothly and applies template replacements every frame.

Quick start

  1. Install the plugin.
  2. Add a Browser Source in OBS and point it to the overlay index.html.
  3. In the Fly Scoreboard dock, select that Browser Source (so the plugin can keep it in sync).
  4. Optional: choose a templates folder, then load any template from the dock dropdown. The selected template becomes the active resources path.
  5. Customize teams, add custom fields (scores), set timers, and go live.
Tip: Keep index.html, style.css, script.js and plugin.json in the same folder. The default overlay runtime connects to ws://127.0.0.1:4457 first and fetches plugin.json by relative path as a fallback.
