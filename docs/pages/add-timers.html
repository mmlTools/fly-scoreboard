Add New Timers

Fly Scoreboard supports multiple timers (halves, periods, power plays, breaks). Your overlay can show any subset.

Timer fields

Each timer entry typically includes:

  • label - display name (e.g., “First Half”)
  • mode - countup or countdown
  • initial_ms, remaining_ms, last_tick_ms
  • running, visible
  • Derived by runtime: live_ms and mmss

In your overlay template

<div class="clock" fs-if="timers[0] && timers[0].visible">
  <div class="clock__label">{{timers[0].label}}</div>
  <div class="clock__time">{{timers[0].mmss}}</div>
</div>

Multiple timers (fixed slots)

<div class="clock" fs-if="timers[1] && timers[1].visible">...</div>
<div class="clock" fs-if="timers[2] && timers[2].visible">...</div>

Best practices

  • Hide unused timers with visible and fs-if.
  • For countup timers, the overlay will display from remaining_ms upward while running.
  • When switching periods, update label and reset initial_ms/remaining_ms.
The overlay fetches plugin.json once per second, but the timer display updates every frame using last_tick_ms. This is why timer text remains smooth.
