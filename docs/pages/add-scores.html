Add New Scores

Use Custom Fields to create additional score rows (e.g., Score, Fouls, Timeouts, Shots, Penalties).

In the dock

  1. Open the Custom Fields (or ‚ÄúScores‚Äù) section.
  2. Click Add to create a new row.
  3. Set the label (example: FOULS).
  4. Adjust the left/right values (home/away), and toggle visibility if needed.

In plugin.json

Each custom field is stored like: 

{
  "label": "Score",
  "home": 0,
  "away": 0,
  "visible": true
}

In your overlay template

The runtime provides fields_xy[] so your layout stays correct when swap_sides is enabled. 

<div class="row" fs-if="fields_xy[0] && fields_xy[0].visible">
  <div class="lbl">{{fields_xy[0].label}}</div>
  <div class="left">{{fields_xy[0].x}}</div>
  <div class="right">{{fields_xy[0].y}}</div>
</div>

Scaling to multiple rows

The templating engine is intentionally simple (no loops). The typical pattern is to author a fixed number of rows and guard them: 

<div class="row" fs-if="fields_xy[1] && fields_xy[1].visible">...</div>
<div class="row" fs-if="fields_xy[2] && fields_xy[2].visible">...</div>
<div class="row" fs-if="fields_xy[3] && fields_xy[3].visible">...</div>
Keep your overlay resilient: always use fs-if="fields_xy[i]" before reading fields_xy[i].x to avoid template flashes if the array is shorter.
